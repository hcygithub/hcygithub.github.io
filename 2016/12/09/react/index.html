我们发现 JavaScript代码里面竟然出现了Html标签，没错这就是JSX。那这种奇怪的语法浏览器能执行吗？当然不能！所以我们要在script标签的type里面写上text/babel，让babel来处理它，例如上面的Demo01中的：

被babel转换后变成

所以说JSX本质上是一种针对React.createElement(component, props, ...children)的 syntactic sugar。出于性能的考虑，JSX代码都要转换成普通的JavaScript代码才放到生产环境中去的。我们学习的话就不管这么多了，怎么简单怎么来，直接引入babel.min.js

JSX语法

1、JSX能嵌入JavaScript表达式

例如上面Demo01中的props.name，JSX中嵌入的JavaScript表达式都要用{}括起来

2、JSX也是一个表达式

这个也很好理解，例如Demo02中的<h1>Hello, {props.name}</h1>

3、JSX指定属性

例如Demo01中<Welcome name="Sara"/>的属性name="Sara"

4、JSX能够包含子标签

类似xml，JSX也能够包含子标签，当无子标签时可以用/>直接关闭

5、JSX能够防止注入攻击

React DOM会对嵌入JSX的任何值进行转义，所以不用担心XSS攻击

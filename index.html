JavaScript简介

历史

1995年，Brendan Eich发明了LiveScript，随后为了利用Java的影响力，Netscape将LiveScript改名为JavaScript。随后微软也推出了JScript，再加上CEnvi推出的ScriptEase，早期的JavaScript三足鼎立，但是没有标准，非常混乱。这种情况非常不利于web的发展，后来ECMA协调各大公司，制定了JavaScript的标准ECMAScript 1，后来各家JavaScript都慢慢实现了ECMAScript的标准。2015年6月17日，ECMAScript 2015（即ES6，以后的官方都采用ECMAScript+年份命名）标准正式发布了。

JavaScript

JavaScript是什么？JavaScript跟ECMAScript不是一个东西，JavaScript包含了：

  • ECMAScript
  • DOM 文档对象模型 DOM让我们能跟Web内容进行交互
  • BOM 浏览器对象模型 BOM让我们能够跟跟浏览器进行交互

组件State与生命周期

再拿之前的时钟的demo说事，我们对这个demo修改一下

这个demo信息量很大，只要把这个demo弄懂了这个小节就ok了。
首先我们看到现在我们只用了一个<Clock/>标签就能让时钟自己动起来，外面没有再包裹tick函数，也没有在外面调用setInterval，因为我们把时钟的状态在Clock组件内部维护起来了。接下来我们把Clock组件你们的函数一个个分析一下：

constructor
组件构造函数，构造函数内部一定要调用super(props);，除此之外函数内部还有个state对象，这个对象是用来维护组件内部的各种状态，一般是在构造函数中初始化

componentDidMount
这是组件的一个生命周期方法，在组件被加载到DOM节点后会立刻被调用，在这个方法里面设置state会重新触发渲染，所以我们在这里起定时器来更新state

componentWillUnmount
这也是一个生命周期方法，这个方法会在组件被移除销毁之前立即调用，一般我们会在这里做一些释放资源的工作，所以我们在这个函数里面清除我们之前设置的定时器

tick
这个是我们自定义的一个函数，我们通过在这里面更新state来刷新UI，大家可能注意到了Do Not Modify State Directly这句注释，意思是不能像这样this.state.date=new Date();更新state，因为这样直接更新并不能重新渲染组件，所以我们必须得用setState来更新state

render
这个函数是必须的，你可以通过他返回你想被渲染的东西，如果你什么都不想渲染，可以返回null或者false

介绍这里我相信大家已经对组件有更深刻的认识了吧，接下来介绍一下组件怎样处理事件。

组件事件处理

这个demo我们发现React组件处理事件跟DOM元素处理事件非常相似，但是他们还是有两个地方不一样

  • React事命名使用驼峰命名法，而不是使用小写
  • 在JSX里面你传递一个函数而不是字符串作为事件处理器

另外大家可能还发现了，在构造函数里面有这样一句话this.handleClick = this.handleClick.bind(this);为什么要这样弄一下，因为在JavaScript中类的方法默认并没有绑定，如果没有这一步，实际调用的时候this.handleClick中的this就是undefined，具体请参考这个文档
